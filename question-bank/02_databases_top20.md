# Топ 20 вопросов по базам данных

---

**1. Что такое B-Tree индекс и как он работает?**
> Сбалансированное дерево. Каждый узел содержит ключи и указатели на дочерние узлы. Листья связаны в список для range-запросов. Поиск O(log n). PostgreSQL использует B-Tree по умолчанию.

**2. Что такое составной индекс и правило левого префикса?**
> Индекс по нескольким колонкам `(a, b, c)`. Работает для запросов начинающихся с левых колонок: `WHERE a=1`, `WHERE a=1 AND b=2`. Не работает: `WHERE b=2`, `WHERE c=3`.

**3. В чём разница между кластерным и некластерным индексом?**
> Кластерный (в PostgreSQL — таблица heap + отдельные индексы): данные не упорядочены физически по индексу. В MySQL InnoDB primary key — кластерный, данные хранятся в порядке PK. Некластерный хранит указатель на строку.

**4. Что такое ACID?**
> **A**tomicity — транзакция либо полностью выполняется, либо полностью откатывается. **C**onsistency — переход из одного валидного состояния в другое. **I**solation — транзакции изолированы друг от друга. **D**urability — закоммиченные изменения сохраняются даже при сбое.

**5. Какие уровни изоляции существуют?**
| Уровень | Dirty Read | Non-Repeatable Read | Phantom Read |
|---|---|---|---|
| Read Uncommitted | ✅ | ✅ | ✅ |
| Read Committed | ❌ | ✅ | ✅ |
| Repeatable Read | ❌ | ❌ | ✅ |
| Serializable | ❌ | ❌ | ❌ |

PostgreSQL по умолчанию: Read Committed.

**6. Что такое Dirty Read, Non-Repeatable Read, Phantom Read?**
> **Dirty Read** — читаем незакоммиченные изменения другой транзакции. **Non-Repeatable Read** — дважды читаем одну строку, получаем разные значения. **Phantom Read** — дважды делаем SELECT, количество строк изменилось.

**7. Как работает MVCC в PostgreSQL?**
> Каждая строка имеет `xmin` (транзакция создавшая) и `xmax` (транзакция удалившая). При UPDATE — старая строка помечается xmax, создаётся новая. Читатели видят снапшот на момент начала транзакции. Не блокируют писателей.

**8. Что такое WAL (Write-Ahead Log)?**
> Журнал изменений. Перед изменением данных — запись в WAL. При сбое — восстановление из WAL. Основа durability и репликации в PostgreSQL.

**9. Что такое EXPLAIN ANALYZE и как читать план?**
> `EXPLAIN ANALYZE` показывает фактический план выполнения запроса с реальным временем. Ключевые показатели: Seq Scan vs Index Scan, rows (estimate vs actual), loops. Высокий cost и Seq Scan на большой таблице — сигнал к оптимизации.

**10. Когда индекс не используется?**
> (1) Функция на колонке: `WHERE LOWER(name) = 'ivan'` — нужен functional index. (2) Неравенство `!=`. (3) LIKE с wildcard в начале `'%text'`. (4) Таблица маленькая — seq scan быстрее. (5) Тип данных не совпадает.

**11. Что такое N+1 проблема?**
> 1 запрос за списком + N запросов за деталями каждого элемента. Решение: JOIN или SELECT ... WHERE id IN (...).

**12. Что такое партиционирование и зачем нужно?**
> Разделение большой таблицы на физические части (по диапазону, списку, хэшу). Ускоряет запросы с фильтром по ключу партиции, упрощает архивирование старых данных.

**13. Что такое репликация PostgreSQL?**
> Streaming Replication: primary пишет в WAL, replica применяет WAL-записи. Синхронная (primary ждёт подтверждения) или асинхронная (не ждёт, возможна потеря данных). Logical Replication — реплицирует изменения на уровне строк, а не WAL.

**14. Чем INDEX SCAN отличается от INDEX ONLY SCAN?**
> INDEX SCAN: индекс находит ctid (указатель на строку), затем читает heap (таблицу). INDEX ONLY SCAN: все нужные данные есть в индексе — heap не читается. Работает если все SELECT колонки покрыты индексом (covering index).

**15. Что такое транзакционный deadlock и как PostgreSQL его обрабатывает?**
> Две транзакции ждут ресурсы друг друга. PostgreSQL обнаруживает цикл ожидания и отменяет одну транзакцию с ошибкой `deadlock detected`. Клиент должен повторить транзакцию.

**16. Что такое оконные функции?**
```sql
SELECT name, salary,
       AVG(salary) OVER (PARTITION BY dept ORDER BY salary) as avg_dept
FROM employees;
```
> Вычисляют значение для каждой строки относительно "окна" (группы строк). Не схлопывают строки как GROUP BY. ROW_NUMBER, RANK, LAG, LEAD, SUM OVER.

**17. Чем DELETE отличается от TRUNCATE?**
> `DELETE` — удаляет строки по условию, записывает в WAL каждую удалённую строку, медленнее. `TRUNCATE` — удаляет все строки одной операцией, быстрее, не может быть с WHERE, сбрасывает sequences.

**18. Что такое connection pooling и зачем нужен PgBouncer?**
> PostgreSQL создаёт отдельный процесс на каждое соединение (~5MB RAM). При большом количестве клиентов (100+) — pgbouncer мультиплексирует их в меньшее число реальных соединений к БД.

**19. Что такое индекс на выражение (functional index)?**
```sql
CREATE INDEX ON users (LOWER(email));
```
> Индекс хранит результат выражения. Используется когда запросы фильтруют по функции от колонки.

**20. Что такое `SELECT FOR UPDATE`?**
> Блокирует выбранные строки на уровне строк (row-level lock) до конца транзакции. Другие транзакции не могут изменить эти строки. Используется для pessimistic locking: "прочитай и заблокируй для изменения".
